# Agent.md

本文件用于约束后续代理在本项目中的工作方式。该项目是机械制造技术基础课程大作业，核心产物是 LaTeX 报告而不是程序代码。

## 项目目标

- 维护并完善 `report.tex`
- 根据 `task1.md`、`task2&3.md`、`add.md` 等来源文件同步报告正文
- 编译生成 `report.pdf`
- 保持工作图、封面、目录、章节结构和参考文献一致

## 文件职责

- `report.tex`：唯一正式报告源文件
- `report.pdf`：编译产物
- `task1.md`：步骤一内容来源
- `task2&3.md`：步骤二、三内容来源
- `add.md`：对现有章节的补充材料
- `template.md`：报告模板和章节结构参考
- `require.md`：作业要求
- `杠杆（CA6140车床）831009.png`：工作图

## 工作原则

1. 先读来源文件，再改 `report.tex`

- 不要凭记忆改报告内容。
- 如果用户说“我更新了 md”，先重新读取对应 Markdown 文件。
- 正文内容以用户提供的 Markdown 为准，除非用户明确要求重写。

2. 内容修改与排版修改分开处理

- 用户要求“原样写入”时，只做 LaTeX 转写，不改写措辞。
- 用户允许“格式优化”时，可以做轻量美化：
  - 去掉多余的 `1）`、`（1）`
  - 改成更自然的 LaTeX 列表
  - 给条目标题加粗
  - 调整表格、段落、封面、目录排版
- 不得擅自增删技术结论、工艺路线或工序内容。

3. 工艺内容处理规则

- “工艺基准的选择”这类章节如果同时存在于多个来源文件，应先整合，再写入 `report.tex`。
- 整合时优先保留：
  - 用户最新补充内容
  - 更具体的工序定位说明
  - 更完整的设计/粗/精基准逻辑
- 整合的目标是“互相补充”，不是二选一覆盖。

4. 机械图样和尺寸表达

- 孔径、公差、粗糙度等符号要用规范 LaTeX 形式表示。
- 对于如 `Φ25(+0.023,0)` 这类尺寸，应优先改为右上/右下公差形式。
- 不要随意改动图样中的尺寸值、孔号、角度、粗糙度要求。

5. 封面与目录

- 封面字段应与标题位于同一页。
- 目录单独位于下一页。
- 若用户给出新的封面格式要求，应优先满足，并避免影响正文页码结构。

## 编译与验证

修改 `report.tex` 后，应至少执行一次：

```bash
xelatex -interaction=nonstopmode -halt-on-error report.tex
```

如目录、交叉引用、页码有变化，应再执行一次：

```bash
xelatex -interaction=nonstopmode -halt-on-error report.tex
```

验证重点：

- `report.pdf` 成功生成
- 封面、目录、正文分页符合用户要求
- 表格未明显溢出
- 中文正常显示
- 图像正常插入

## 禁止事项

- 不要删除用户提供的 Markdown 来源文件
- 不要把草稿内容写回来源 Markdown，除非用户明确要求
- 不要擅自重构整篇报告结构
- 不要把“排版修正”变成“内容重写”
- 不要在没有读取最新文件的情况下覆盖 `report.tex`

## 输出偏好

- 优先给出直接结果，不做空泛解释
- 说明修改时，优先说清：
  - 改了什么
  - 是否重新编译
  - 产物文件在哪里
